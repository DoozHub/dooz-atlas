# Model Behavior Notes

Observations about specific AI model behaviors and quirks.

---

## GPT-4o

### Strengths
- Good general-purpose coding
- Strong at following patterns
- Fast response times

### Quirks
- Sometimes over-engineers solutions
- Can be verbose in explanations
- Occasionally insists on deprecated patterns

### Best For
- Standard implementation tasks
- Code explanation
- Documentation

---

## GPT-4 o1 / o1-pro

### Strengths
- Excellent at complex reasoning
- Good at multi-step problem solving
- Strong at trade-off analysis

### Quirks
- Expensive (use sparingly)
- Can overthink simple tasks
- Long thinking time for simple queries

### Best For
- Architecture decisions
- Security analysis
- Complex refactoring

## Local Models (Qwen, Llama)

### Strengths
- Cost-effective at scale
- Data stays local
- No rate limits

### Quirks
- Quality varies by size
- May miss context nuances
- Less consistent than cloud models

### Best For
- Mechanical tasks
- High-volume operations
- Sensitive data processing
